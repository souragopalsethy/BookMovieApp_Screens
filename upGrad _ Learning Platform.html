Project Overview

Important Note

 

This is going to be a group assignment where you would be part of a group of 3-4 learners. You and your group members are supposed to come up with the final solution for this assignment. The members of a particular group are expected to do the task division and identify the smaller subtasks for this assignment and divide the work among themselves. This exercise ensures that you are well-equipped to work in a collaborative environment in your real-time jobs as well. Final grading will also be done for the group (not an individual).

 

This project is graded. Hence, you are strongly advised not to discuss the solution/solution approach with your peers. You are also strongly advised to refrain from publishing the solution of this project on any public platform such as GitHub etc. If anyone is found violating these conditions, a severe penalty will be imposed as per the IIITB Guidelines.

 

Since this is a group project, you might need to work in a collaborative environment on git to ensure that each group member can contribute to git. In that case, you all can add your code on git (with visibility of the repo as private) and add other team members as collaborators. This will ensure that other people on the internet cannot access your solution, and at the same time, your team members are able to push and pull the changes on the GitHub repo of the project solution.

 

About the Project

This project is aimed at developing the front end of a real-world application in React. With this application, which is named BookMyMovie, users can browse upcoming and released movies; filter released movies based on certain parameters; and view details such as genre, artists and trailer of released movies.

Also, registered users can book a show for a released movie. In the corporate world, you are required to work on projects where you cannot build everything from scratch. If you do that, it is more like you are reinventing the wheel.

In this project as well, you will not be building each and every component from scratch while building the Movies application. Instead, you will be leveraging Material-UI, which implements Google’s Material Design language and helps with building React components. You will be customising these readily available components as per your needs and will build the entire home page of the application, which includes a header showing the logo of the application, and the register and login functionality.

On the home page, you will also learn how to display an upcoming movies grid, a released movies grid, and filters that can be applied to a released movie. We will be providing you with the Book Show and the confirmation page in advance. You can see and learn from them, and you have to integrate them into your project.


Important: Throughout the project, on several occasions, you will have to perform online searches to solve issues. This is especially important because, in a real software development environment, engineers are not expected to know everything in advance. Instead, they are expected to gradually discover ways to solve problems through their own research, trial and error, and by searching online. So, it is important that you utilise your own trials as well as search for solutions online in order to be able to create the project successfully.

      Goal

      While building the movie application, you can put into practice the skills that you learnt and the knowledge that you gathered in the module on ReactJS. You will be provided with certain guidelines, which will help you build this application. These guidelines will provide you with proper instructions on how you should create the front end of the application using ReactJS and integrate it with the back end. This will help you practise sending an HTTP request to the server and getting an HTTP response back from it. The response data that you obtain from the server can, then, be manipulated as per the application requirements and be used to render vital information on your application.

      Overall, developing this application will help you understand how real-world front-end applications are developed while following the best software development practices.

      After you have developed this application, you will feel confident in building the front-end user interface using React and integrating it with the back end for any real-world application.

       

      Version Control Best Practices

      You must develop this application ‘individually’.

      You should create a master repository for the application and push the initial code stub (starter code provided to you at the start of the next segment) to this master repository. 

      Please follow these best practices when using Git and GitHub to strive towards becoming a better software engineer:

      • Commit your code often and frequently

      • Make small and incremental commits

      • Before committing your code, make sure it works

      • Write brief and relevant commit messages
